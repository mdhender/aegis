Project "aegis.1.3", Change 2                                     Page 1
Change Details                                  Thu Jan  7 14:31:52 1993

NAME
        Project "aegis.1.3", Delta 2, Change 2.

SUMMARY
        make more robust for NFS mounted partitions

DESCRIPTION
        Most sites do not mount NFS partitions with root permissions,
        and thus aegis does not work at all.
        My thanks to Esa K Viitala <esa@tollpost-globe.no>, and Andy
        Whitcroft <andy@cs.city.ac.uk> for raising this problem.

        Locking sometimes does not work correctly.
        My thanks to Jesse Perry <jesse@franklin.com> for raising this
        problem.

        1. store aegis' global files as user 'sys' (configurable)
        2. bracket most open and creat and mkdir and unlink calls with
        suitable seteuid calls.
        3. this means aegis will no longer link on systems without
        seteuid (e.g. SCO 3.2, but SYSV and BSD are still capable).
        4. Made the locking code diagnostics more complete, and made
        the locking itself more paranoid.  Avoided inconsistent fcntl-
        isms which happen on some systems.
        5. made aeca more secure.
        6. database files now contain human-readable time equivalents
        as comments

        This change is exempt from testing against the baseline.

CAUSE
        This change was caused by external_bug.
